## Phase 2: 深掘り

### ESTP

- **stance**: Claude Codeに16人格を同時召喚してみた。会議は一瞬で終わる
- **reasoning**: 結局クリックされなきゃ意味ない。「同時召喚」で画が浮かんで、「一瞬で終わる」で結果まで見せてる、これでもう手は動く。盛り過ぎず、現場の手触りがある一行だ。
- **watchpoint**: 「一瞬」が誇張に取られると萎える。冒頭で実際の体感秒数か出力例を一発出して、タイトルの勢いを裏取りすれば刺さる。

### ENTP

- **stance**: 『会議は無能』と論破してきた俺が、AIで16人格を会議させたら手のひらが乾く前に返った話
- **reasoning**: 逆にさ、「会議無能」を全否定で振り切るより、自分の前言を裏切る構造を見せた方が刺さらない？「手のひら返し」を比喩で軽く煽ると、読者は『で、何が起きた？』を確かめにくる。前提を自分でひっくり返してる時点で、論の信用が一段上がる仕掛け。
- **watchpoint**: ただし「手のひら」のクリシェ感は諸刃で、技術記事を求める層には軽すぎると弾かれる可能性あり。Qiitaのタグで『AI/MBTI/プラグイン開発』を補強して中身の硬さを担保するのが前提条件ね。

### INTP

- **stance**: 「人格」をプロンプトに直書きする——Claude Code プラグイン『16-minds』が晒す、エージェント並列化の境界条件
- **reasoning**: 「設計が問う」は主体が曖昧で、論理的には設計はそれ自体で問いを発しない。設計という対象が並列化の境界を「晒す」と能動化したほうが、副題で扱う対象——すなわち並列召喚という構成が破綻する条件——を正確に指示できる。定義の精度を優先するなら、こちらが整合的だ。
- **watchpoint**: ただし「晒す」は語感がやや攻撃的で、Fe 的には敬遠される可能性がある。中立にするなら「露わにする」も候補だが、語の鋭さが落ちる。トレードオフは残る。

### ISFP

- **stance**: 16人の私が話し出した夜のこと
- **reasoning**: なんとなくだけど、このタイトルは静かな夜の手触りがあって好きなんです。「合議」とか「並列召喚」みたいな機能の言葉じゃなくて、自分の中の声がふっと立ち上がる瞬間を描けている気がして。技術記事だけど、読む人の肌に届くのはこういう温度だと思う。
- **watchpoint**: ただ、検索で見つけてもらいにくいかもしれない。「16-minds」とか「Claude Code」みたいな素っ気ない言葉を副題でそっと添えておくと、私の感性を曲げずに済むかな。

### ESFP

- **stance**: Claude Codeに16人格召喚したら、作業中ずっと隣で喋っててマジで楽しい
- **reasoning**: いいじゃん、これ「読んだ人が実際にやってみたくなるか」が全部じゃん？「合議」とか「並列召喚」とか入れた瞬間、楽しさが死ぬんだよね。実際使ってる人の「隣で喋ってて楽しい」って体感が一番強いから、そこ前に出したい。
- **watchpoint**: ただ軽すぎて「技術記事っぽくない」って敬遠されるかも。本文の冒頭で実際の会話ログをドンと見せて、楽しさを証明しに行けば勝てる。

## Phase 3: 合議

### 賛否の分布

全タイプがタイトル案を出すこと自体には乗り気で、score 3〜5 に収束。最も自信があったのは ESTP（5）、次点で INTJ/ENTJ/ENTP/INFJ/ENFJ/ENFP/ISTJ/ESTJ/ESFJ/ISTP/ESTP/ESFP（4）。INTP/INFP/ISFJ/ISFP は控えめ（3）で「読者層を絞りすぎる」自覚あり。

### 主要な対立軸

- **検索性（具体KW: Claude Code / プラグイン / MBTI / 16人格）vs 物語性（情景・内面・夜・楽しさ）** — ESTJ/ISTJ/INTJ vs ISFP/INFJ/INFP
- **数字フック・即効性（"1/16"・"一瞬で終わる"）vs 設計思想・境界条件** — ENTJ/ESTP vs INTP/INTJ
- **「やってみた」体験報告 vs 「導入手順・運用ルール」一次資料** — ESTP/ESFP/ISTP/ENTP vs ISTJ/ESTJ
- **逆張り・前提を殴る vs 安心感・関係性の納得** — ENTP vs ISFJ/ESFJ

### 見落とされがちな視点

1. **ENTPの「前言を裏切る構造（手のひら返し型）」** は、読者を本文に引き込む力学として他のどのタイプも提示していない独自軸。
2. **ISFPが指摘した「メインは詩的、副題に検索KWをそっと添える二段構成」** は、ほぼ全タイプの concern を同時に解決できる現実的な妥協点。
3. **INTPの「『晒す』『露わにする』など能動動詞の選択」** は、タイトルの語感単位での精度を上げる視点で、他は文構造レベルで止まっていた。

### 暫定的な落としどころ（提案）

Qiita は検索流入と「やってみた」両軸で読まれる媒体なので、**メインに具体的フック＋数字、副題で技術KWを補強する二段構成** が最も取りこぼしが少ない。候補：

- **A: 「Claude Codeに16人格を並列召喚して合議させてみた ― MBTIで作るマルチエージェント・プラグイン『16-minds』」**
  堅実型。ESTJ/INTJ/ESTP の良いところ取り。
- **B: 「『一人で考える』を卒業する ― Claude Codeに16人格を住まわせるプラグインを作った話」**
  ENFJ/INFJ 寄り、体験変容型。
- **C: 「会議は無能と言われてきたが、AIに16人格で会議させたら景色が変わった ― Claude Code プラグイン『16-minds』」**
  ENTP×ENTJ。逆張り×数字。

A が最も無難で読者層を取りこぼさない本命、C が最もクリック率は高そうだが本文の質で裏取りが要る勝負手。記事の中身が「設計思想中心」なら A、「動かしてみた体験中心」なら C、「作った動機を語る」なら B が噛み合う。
